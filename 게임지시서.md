## 2. 소스 파일 역할 정리

### index.html
게임의 핵심 파일. 별도 프레임워크 없이 순수 HTML/CSS/JS로 구성.

| 섹션 | 역할 |
|------|------|
| `<style>` | 전체 게임 UI 스타일 (CSS 변수, 타이틀·HUD·그리드·오버레이) |
| Cat Sprite | 고양이 스프라이트 시트 렌더링 (Canvas 2D, 프레임 애니메이션) |
| Dragon Canvas | 드래곤 캐릭터 직접 드로잉 (Canvas 2D, 절차적 생성) |
| Debug Logger | 게임 이벤트 로그 패널 (`DEBUG_LOG_ENABLED = true`로 활성화) |
| 자동 힌트 | 3초 무입력 시 최적 이동 방향 자동 하이라이트 |
| Game State | 점수·레벨·타이머·보드 상태 관리 |
| Board Engine | 타일 생성·렌더·스왑·매치·낙하·채우기 |
| Drag System | 마우스/터치 드래그 스왑 처리 |
| AudioManager | 사운드 풀 관리, BGM/SFX 제어, 뮤트 처리 |
| Title Screen | 타이틀 화면 (별 파티클, 고양이 셀레브레이션, Google 로그인) |
| Settings | 소리 토글 (localStorage 저장) |

### config/assets.js
모든 에셋 경로를 `window.ASSETS_CONFIG`에 집중 관리.  
파일 이동 시 이 파일만 수정하면 됨.

### config/levels.js
`window.LEVEL_CONFIG`로 노출. 레벨 전체 난이도 곡선 제어.

### config/sounds.js
`window.SOUNDS_CONFIG`로 노출. 각 사운드 ID, 파일명, 볼륨, 루프 여부 정의.

### config/items.js
`window.ITEMS_CONFIG`로 노출. 폭탄·번개·무지개 아이템 정의 (현재 미구현, 확장 예약).

### config/specialTiles.js
특수 타일 타입의 **설계 문서 역할**. 트리거 조건·효과 설명 포함.  
실제 로직은 `js/specialTiles.js`와 `js/specialTileEffects.js`에 구현.

### css/specialTiles.css
특수 타일 5종의 외형(글로우·테두리)과 발동 애니메이션 클래스 정의.

### js/specialTiles.js
- `SPECIAL_TYPES` 상수 (board 값 10~14)
- `isSpecialTile(value)` — board 값이 특수 타일인지 판별
- `getSpecialTileClass(value)` — CSS 클래스명 반환
- `getSpecialTileType(matchedCells)` — 매치 형태 분석 → 생성할 특수 타일 타입 반환
- `timebombCounters` + `setTimebomb` / `tickTimebombs` / `clearTimebomb` — 타임밤 카운트다운 관리

### js/specialTileEffects.js
특수 타일 발동 시 **제거 대상 셀 목록을 계산**하는 순수 함수 모음.  
실제 제거·애니메이션은 `processMatches()`가 담당.

| 함수 | 역할 |
|------|------|
| `getDrillCells` | 지정 행 또는 열 전체 반환 |
| `getBlackholeCells` | 중심 반경 2칸 원형 범위 반환 |
| `getChainCells` | 같은 타입 최대 8개 랜덤 선택 반환 |
| `getTimebombCells` | 중심 반경 3칸 원형 범위 반환 |
| `getMagnetCells` | 같은 타입 전체 (개수 제한 없음) 반환 |

### js/supabase-client.js
Supabase JS SDK를 CDN으로 로드한 뒤 클라이언트 인스턴스를 `window.supabaseClient`에 설정.

### js/auth.js
| 함수/변수 | 역할 |
|-----------|------|
| `window._cachedUser` | 현재 로그인 유저 캐시 (네트워크 없이 즉시 참조) |
| `loginWithGoogle()` | Google OAuth 리다이렉트 로그인 |
| `logout()` | 세션 종료 |
| `getCurrentUser()` | 현재 로그인 유저 반환 (서버 검증, 네트워크 요청 있음) |
| `onAuthReady(callback)` | 인증 상태 변경 시 콜백 실행 + 미전송 데이터 동기화 트리거 |

> **`_cachedUser` 사용 원칙**: 게임 플레이 중 유저 참조는 반드시 `_cachedUser` 사용.  
> `getCurrentUser()`는 네트워크 요청이 있어 실패 가능성 있음.

### js/leaderboard.js
| 함수 | 역할 |
|------|------|
| `saveScore(score)` | 로그인 유저의 점수를 `scores` 테이블에 INSERT |
| `getMyBestScore()` | 현재 유저의 최고 점수 조회 |
| `getLeaderboard()` | 상위 20명 리더보드 조회 (`leaderboard` VIEW) |
| `saveUserProgress(userId, data)` | `user_progress` upsert (로컬 캐시 후 Supabase 전송) |
| `loadUserProgress(userId)` | `user_progress` 조회, 실패 시 로컬 캐시 반환 |
| `saveLevelScore(userId, level, score)` | `level_scores` insert (로컬 캐시 후 Supabase 전송) |
| `getProgressLeaderboard()` | max_cleared_level 기준 상위 20명 |
| `getMyRank(userId)` | 내 순위 숫자 반환 |
| `flushPendingSaves()` | 로컬 큐에 쌓인 미전송 데이터를 Supabase로 전송 |
| `checkSupabaseConnection()` | DB 연결 상태 확인 후 디버그 로그에 출력 |
| `recordLogin(user)` | `login_history` 테이블에 접속 기록 (1시간 중복 방지) |

### js/version.js
```js
window.APP_VERSION = '2026-06-24 #8';
```
- 푸시할 때마다 날짜와 번호를 수동으로 업데이트
- 디버그 LOG 버튼, 설정 패널 하단, 로그 패널 첫 줄에 표시

## 3. 게임 핵심 메커니즘

### 기본 흐름
```
타이틀 화면 → 게임 시작 → 레벨 로딩 → 게임 플레이 → 클리어/실패 → 다음 레벨 or 재시작
```

### 보드 규칙
- 타일을 인접한 칸과 **스왑(클릭 2회 또는 드래그)**
- 3개 이상 같은 타입이 가로·세로로 이어지면 **매치 → 제거 → 낙하 → 채우기**
- 매치 성공 시 고양이가 탈출구 방향으로 한 칸 전진
- 지정 매치 수를 달성하면 **레벨 클리어**
- 제한시간 0초가 되면 **게임 오버**

### 입력 처리
| 조작 | 동작 |
|------|------|
| 타일 클릭 | 1회: 선택 / 인접 타일 클릭: 스왑 |
| 드래그 | 이동 거리 ≥ 6px(DRAG_THRESH) 시 드래그 인식, 방향 최소 거리 18px(DIR_MIN) |
| 터치 | 터치스타트/무브/엔드 이벤트로 동일 처리 |

### 점수 계산
```
기본: 매치된 타일 수 × 10점
콤보(5개+): 위 결과 × 2 (더블)
연쇄 매치(1초 이내 연속): comboMatching 사운드 재생
```

### 자동 힌트
- 마지막 입력 후 **3,000ms** 경과 시 최적 이동 방향 하이라이트
- 유효한 이동이 없으면 자동 셔플 실행

### 위험도 시스템 (Urgency)
```
urgency = 1 - (남은시간 / 기본시간)
드래곤 위치 = 2% + urgency × (고양이 위치 - 2%)
위험도 바 너비 = urgency × 100%
```
- urgency가 1에 가까울수록 드래곤이 고양이 위치까지 수렴

## 4. 레벨 설계 수치

### 전역 설정 (config/levels.js)

| 항목 | 값 |
|------|----|
| 최대 레벨 | 100 |
| 기본 제한시간 | **60초** |
| 타일 종류 수 | **5종** |
| 동물 배치 시드 | 98765 (결정론적 랜덤) |

### 그리드 티어

| 티어 | 레벨 범위 | 그리드 크기 | 레벨당 추가시간 |
|------|-----------|-------------|-----------------|
| 초급 | 1 ~ 30    | **8 × 8**   | 5초 (미구현)    |
| 중급 | 31 ~ 100  | **10 × 10** | 3초 (미구현)    |

> `timePerLevel`은 config에 정의되어 있으나 현재 게임 로직에서 시간 추가는 비활성화 상태.

### 레벨별 필요 매치 수

```
총 필요 매치 = 10 + min(floor(레벨 / 5), 10)
```

| 레벨 구간 | 필요 매치 |
|-----------|-----------|
| 1 ~ 4     | 10회      |
| 5 ~ 9     | 11회      |
| 10 ~ 14   | 12회      |
| 25 ~ 29   | 15회      |
| 50+       | 20회 (최대) |

### 출구 문 앞 추가 장벽 수

```
추가 장벽 = min(floor(레벨 / 5), 10)
```
- 기본 10보 외에 추가로 제거해야 할 장벽 아이콘으로 시각화

### 고양이 이동 경로

```
시작 위치: 54% (화면 좌측 기준)
최대 위치: 83% (탈출 직전)
클리어 위치: 92% (탈출 성공)

이동 공식: 54 + (완료된 기본 보수 / 10) × (83 - 54)
```

### 타이머 경고 이벤트

| 잔여 시간 | 이벤트 |
|-----------|--------|
| 20초      | 고양이 `scare` 상태 전환 + `hurryUp` 사운드 1회 |
| 10초      | `hurryUp` 사운드 2회째 |
| 0초       | 게임 오버 |

### 타이머 링 색상

| 잔여 비율 | 색상 |
|-----------|------|
| 40% 이상  | `#5ddbff` (파랑) |
| 20% ~ 40% | `#f5c842` (노랑) |
| 20% 미만  | `#ff4400` (빨강) |

## 5. 특수 타일 시스템

### Board 값 규칙

| board 값 | 의미 |
|----------|------|
| -1       | 빈칸 |
| 0 ~ 4    | 일반 타일 (동물 타입 인덱스) |
| 10       | 드릴 (DRILL) |
| 11       | 블랙홀 (BLACKHOLE) |
| 12       | 체인 (CHAIN) |
| 13       | 타임밤 (TIMEBOMB) |
| 14       | 자석 (MAGNET) |

### 생성 조건 (판별 우선순위 순)

| 우선순위 | 매치 형태 | 생성 타일 | 이모지 | CSS 클래스 |
|----------|-----------|-----------|--------|------------|
| 1 | 6개 이상 또는 5개 십자(+) | 블랙홀 | 🌀 | `tile-blackhole` |
| 2 | 5개 일렬 | 체인 | ⚡ | `tile-chain` |
| 3 | 5개 T자·L자 | 타임밤 | 💣 | `tile-timebomb` |
| 4 | 4개 일렬 | 드릴 | ⚙️ | `tile-drill` |
| 5 | 4개 ㄱ·ㄴ자 | 자석 | 🧲 | `tile-magnet` |
| — | 3개 이하 | 없음 (일반 매치) | — | — |

### 특수 타일 효과 상세

#### 드릴 (⚙️ DRILL, board=10)
- **발동**: 스왑 시 즉시 또는 매치에 포함 시
- **효과**: 생성된 방향(행 또는 열) **전체 제거**
- **범위**: 가로 전체 (GRID 너비) 또는 세로 전체 (GRID 높이)
- **방향 결정**: 매치 셀들이 같은 행이면 row, 같은 열이면 col
- **CSS 글로우**: `orange`

#### 블랙홀 (🌀 BLACKHOLE, board=11)
- **발동**: 스왑 시 즉시 또는 매치 포함 시
- **효과**: 중심 기준 **반경 2칸 원형** 범위 모두 제거
- **최대 제거 수**: 반경 2 원형 = 최대 13칸
- **애니메이션**: `bh-pull` (회전하며 0.05배로 수축)
- **CSS 글로우**: `purple`

#### 체인 (⚡ CHAIN, board=12)
- **발동**: 스왑 시 즉시 또는 매치 포함 시
- **효과**: 같은 타입 타일 **최대 8개 무작위 선택** + 자기 자신 제거
- **총 제거**: 최대 9칸 (8개 + 자신)
- **특수 타일이 체인 위치인 경우**: 보드에서 가장 많은 타입으로 자동 선택
- **시각 연출**: SVG 번개 선이 선택된 타일로 연결됨 (0.4초 후 사라짐)
- **CSS 글로우**: `cyan`

#### 타임밤 (💣 TIMEBOMB, board=13)
- **생성 시**: 카운트다운 **3턴**으로 등록 (`data-countdown="3"`)
- **자동 발동**: 3턴 경과 후 자동 폭발
- **즉시 발동**: 스왑 또는 매치에 포함 시 즉시 폭발
- **폭발 범위**: 중심 기준 **반경 3칸 원형** 범위 제거
- **최대 제거 수**: 반경 3 원형 = 최대 29칸
- **카운트 시각화**: 우상단 숫자 표시 (`::after` 의사요소)

| 카운트 | 효과 |
|--------|------|
| 3      | 약한 빨강 글로우 `0 0 8px 2px red` |
| 2      | 중간 글로우 + 0.8초 펄스 애니메이션 |
| 1      | 강한 글로우 + 0.3초 빠른 펄스 애니메이션 |

- **CSS 글로우**: `red`

#### 자석 (🧲 MAGNET, board=14)

## 8. 백엔드 (Supabase)

### 프로젝트 정보
| 항목 | 값 |
|------|----|
| 서비스 | Supabase (PostgreSQL + Auth + RLS) |
| Project URL | `https://krttjnshqvvgogmocnhw.supabase.co` |
| Auth 제공자 | Google OAuth 2.0 |
| 리다이렉트 | `window.location.href` (현재 URL) |

### 오프라인 저장 전략
- 레벨 클리어 시 **localStorage에 즉시 저장** (`rc_pending_saves` 키)
- Supabase 전송 성공 시 큐에서 제거
- 전송 실패(네트워크 오류, Supabase 휴면 등) 시 큐에 보관
- **동기화 트리거**: 페이지 로드(2초 후), 로그인 이벤트, `online` 이벤트
- `checkSupabaseConnection()` 으로 연결 상태를 디버그 로그에서 확인 가능

> **Supabase 무료 플랜 주의**: 1주일 이상 비활성 시 프로젝트 자동 휴면.  
> 휴면 중에는 API 호출 실패 → 오프라인 큐에 누적 → 복구 후 자동 동기화.

## 12. WebView 대응

### 문제
카카오톡 인앱브라우저(WebView)에서 Google OAuth 시도 시 구글이 차단함.  
→ `403: disallowed_useragent` 오류 (Google이 2021년부터 WebView OAuth 전면 차단)

### 해결 코드 위치
`index.html` `<head>` 최상단 첫 번째 `<script>` 블록

### 동작 방식
| 기기 | 동작 |
|------|------|
| Android 카카오톡 | `intent://` 스킴으로 Chrome 강제 오픈 (자동) |
| iOS 카카오톡 | 상단 빨간 배너 표시 → `···` → `Safari로 열기` 안내 |
| 일반 브라우저 | 아무 동작 없음 (정상 진행) |

### 감지 조건
```js
/KAKAOTALK/.test(navigator.userAgent)
```

### Google OAuth "Access blocked" 오류 (별개 문제)
- WebView 문제가 아닌 **Google Cloud Console OAuth 앱이 테스트 모드**인 경우 발생
- 해결: Google Cloud Console → OAuth 동의 화면 → **앱 게시** 또는 테스트 사용자 추가
- `email`, `profile` 스코프만 사용 시 Google 심사 없이 바로 게시 가능
