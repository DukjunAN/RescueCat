## BUG-001 · 특수타일 오생성 (잘못된 타입 + 잘못된 위치)

### 심각도
🔴 Critical — 게임 플레이 근간 붕괴

### 현상
- 5개 일렬 매치 시 CHAIN이 생성되어야 하는데 BLACKHOLE이 생성됨
- 생성된 특수타일 위치가 매치 중심이 아닌 엉뚱한 위치에 나타남

### 스크린샷 재현 케이스
```
row 5:  🟢🟢🟢🟢🟢  ← 5마리 가로 일렬 (→ CHAIN이어야 함)
row 6:  .. .. .. 🌀  ← BLACKHOLE이 여기 생성됨 (❌ 잘못된 타입 + 위치)
```

### 원인

**[원인 1] `findMatches()` 가 모든 타입의 매치를 하나의 배열로 합산**

파일: [index.html](index.html) 라인 1603-1616

```javascript
function findMatches() {
  const s = new Set();
  // 가로 스캔 (type A, B, C 구분 없이 전부 같은 Set에 추가)
  for (let r ...) for (let c ...) { ... s.add(`${r},${c+i}`); }
  // 세로 스캔 (마찬가지로 전부 합산)
  for (let c ...) for (let r ...) { ... s.add(`${r+i},${c}`); }
  return [...s].map(...);  // 타입 혼합된 단일 배열 반환
}
```

동시에 여러 매치가 발생하면 **다른 동물 타입의 셀들이 모두 하나의 배열로 합쳐짐**.

```
예시:
  row 5 초록 고양이 5마리 가로 = 5셀
  row 6 주황 고양이 3마리 별도 = 3셀
  findMatches() 반환 = 8셀 (타입 혼합)
  getSpecialTileType(8) → n≥6 → BLACKHOLE  ← 틀림
  정답: 초록 5마리 → CHAIN, 주황 3마리 → 일반 제거 (별도 처리)
```

**[원인 2] 피벗 위치를 배열 인덱스 중간값으로 선택**

파일: [index.html](index.html) 라인 1571

```javascript
const pivot = m[Math.floor(m.length / 2)];
```

합산된 배열의 중간 인덱스 = 기하학적 중심이 아님.  
타입이 다른 셀들이 섞인 배열에서는 완전히 엉뚱한 위치를 가리킴.

### 수정 방안

`findMatches()`를 타입별로 그룹 분리해서 반환하도록 변경:

```javascript
// 수정 후 반환 형태
function findMatchGroups() {
  // 타입별 Map: { tileType → Set<"r,c"> }
  const groups = new Map();
  // ... 스캔 시 groups.get(t) 에 추가
  return [...groups.values()].map(s =>
    [...s].map(k => { const [r,c]=k.split(',').map(Number); return {r,c}; })
  );
  // 반환: [ [{r,c},...], [{r,c},...], ... ]  타입별 배열 리스트
}
```

호출부에서 각 그룹별로 `getSpecialTileType()` 독립 호출.  
피벗은 각 그룹 내 셀들의 **평균 row/col** (기하학적 중심)으로 결정.

## BUG-002 · 특수타일 피벗 위치 오류 (단일 매치 시에도 발생)

### 심각도
🟠 Major — 특수타일 위치가 항상 틀릴 가능성

### 현상
- 5개 십자 매치 시 중심 셀에 BLACKHOLE이 생겨야 하는데 다른 위치에 생성됨

### 원인

`findMatches()` 의 Set 삽입 순서:
1. 가로 스캔 먼저 → 가로 셀들이 앞에 배치
2. 세로 스캔 나중 → 세로 셀들이 뒤에 배치

```
십자 매치 예시 (중심: row6,col3):
  가로: (6,1),(6,2),(6,3) 먼저 삽입
  세로: (5,3),(7,3) 나중 삽입
  m = [(6,1),(6,2),(6,3),(5,3),(7,3)]
  pivot = m[2] = (6,3)  ← 이 경우는 우연히 맞음

  그러나 가로 시작 위치에 따라:
  m = [(6,2),(6,3),(6,4),(5,3),(7,3)]
  pivot = m[2] = (6,4)  ← 중심(6,3) 아님 ❌
```

### 수정 방안

피벗을 배열 인덱스가 아닌 **평균 좌표**로 선택:

```javascript
// 현재 (잘못됨)
const pivot = m[Math.floor(m.length / 2)];

// 수정
const avgR = Math.round(m.reduce((s, c) => s + c.r, 0) / m.length);
const avgC = Math.round(m.reduce((s, c) => s + c.c, 0) / m.length);
const pivot = m.find(c => c.r === avgR && c.c === avgC)
           ?? m[Math.floor(m.length / 2)];  // fallback
```

## BUG-003 · 초기 보드 매치 미처리

### 심각도
🟡 Minor — 매 게임 시작 시 재현 가능

### 현상
- 게임 시작 시 보드에 3개 이상 동일 타일이 일렬로 배치된 상태로 시작되는 경우 있음
- 유저 입력 없이도 이미 매치된 상태인데 처리되지 않음

### 원인
보드 초기화 시 `findMatches()` 검사 없이 `rnd()`로 랜덤 타일을 채움.

### 수정 방안
보드 초기화 후 매치가 없을 때까지 재배치:

```javascript
function initBoard() {
  do {
    for (let r=0; r<GRID; r++)
      for (let c=0; c<GRID; c++)
        board[r][c] = rnd();
  } while (findMatches().length > 0);
}
```
