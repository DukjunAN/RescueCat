# 특수타일 인터랙티브 튜토리얼 설계서

> 작성일: 2026-06-26  
> 현재 상태: **비활성화** (index.html 튜토리얼 훅 주석 처리됨)

---

## 1. 목적

특수타일이 최초로 생성되는 순간, 해당 타일의 **생성 방법**과 **사용 방법**을 인터랙티브하게 안내한다.  
각 타일 유형별로 딱 한 번만 표시하고, 이후에는 다시 나오지 않는다.

## 3. 튜토리얼 흐름 (2단계)

```
특수타일 생성
     ↓
[Phase 1: 생성 설명] ← 약 500ms 자동 표시
  - chase-scene 영역에 자막 표시
  - 생성된 특수타일에 골드 스팟라이트
  - 나머지 타일 딤 처리
     ↓ (500ms 후 자동 전환)
[Phase 2: 직접 사용] ← 유저가 직접 스왑
  - "이 타일을 스왑해보세요" 안내 자막
  - 특수타일 + 스왑할 인접 타일 하이라이트
  - 👆 손가락 드래그 애니메이션 표시
  - 유저가 허용된 셀만 스왑 가능 (다른 터치 차단)
     ↓ (스왑 완료 후 1.5초)
[완료 메시지] → "확인!" 버튼 → 튜토리얼 종료
```

## 4. UI 구성 요소

### 4-1. `#tut-overlay` (z-index: 9000)
- `position: fixed; inset: 0; pointer-events: none`
- 퍼즐 그리드 위에 오버레이
- 자식 요소:
  - `#tut-spotlight`: 골드 테두리 + box-shadow로 나머지 영역 딤 효과 (pulse 애니메이션)
  - `#tut-finger`: 👆 드래그 방향 애니메이션 (가로/세로)

### 4-2. `#tut-caption` (z-index: 9500)
- `position: fixed; left: 0; right: 0`
- **chase-scene 영역** 위에 동적으로 배치 (`_positionCaption()`으로 getBoundingClientRect 사용)
- chase-scene에 직접 넣지 않는 이유: `overflow: hidden`으로 잘림 발생
- 자식 요소: icon, title, desc, range(패턴 다이어그램), hint, 건너뛰기 버튼

### 4-3. 타일 클래스
| 클래스 | 적용 대상 | 효과 |
|--------|-----------|------|
| `.tut-active-tile` | 하이라이트 타일 | brightness(1.4) + gold glow + pointer-events auto |
| `.tut-inactive-tile` | 나머지 타일 | brightness(0.25) + pointer-events none |

## 7. 핵심 구현 포인트

### 7-1. 타이머 일시정지 (중요)
```javascript
_pauseGame() {
  // animating은 절대 true로 만들지 않음
  // animating = true → index.html onPointerDown에서 모든 터치 차단됨
  clearInterval(window.timerInterval);
}
```

### 7-2. 자막 위치 동적 계산
```javascript
_positionCaption() {
  const rect = document.getElementById('chase-scene').getBoundingClientRect();
  cap.style.top    = rect.top + 'px';
  cap.style.height = rect.height + 'px';
}
```

### 7-3. 스팟라이트 딤 효과
- backdrop element 없음 — `#tut-spotlight`의 `box-shadow: 0 0 0 9999px rgba(0,0,0,0.82)`만으로 구현
- backdrop이 있으면 터치 이벤트가 막힘

### 7-4. 중복 표시 방지

## 8. index.html 연동 위치

| 위치 | 코드 | 설명 |
|------|------|------|
| 일반 특수타일 생성 후 | `tutorialManager.checkAndShow(specialType, m)` | 매치로 특수타일 생성 시 |
| 조합 특수타일 생성 후 | `tutorialManager.checkAndShow(comboResult, [...])` | 스왑 조합으로 생성 시 |
| 스왑 완료 후 | `tutorialManager.onSwapComplete(r1,c1,r2,c2)` | Phase1→2 전환 및 Phase2 완료 감지 |
| 설정 패널 | 튜토리얼 초기화 버튼 | localStorage 초기화 (로컬 테스트용) |

> **현재 상태**: 위 훅들이 모두 `/* ... */` 주석으로 비활성화되어 있음

## 9. 미해결 버그 (재활성화 시 확인 필요)

### 버그 A: Phase 1 표시 시간 문제
- **증상**: Phase 1이 너무 짧게 표시되고 Phase 2로 넘어감
- **원인**: `checkAndShow()` 호출 직후 같은 함수 내에서 `onSwapComplete()`가 동기적으로 호출됨 → 500ms 타이머가 즉시 시작
- **해결 방향**: Phase 1에서 타이머를 더 길게 주거나, "다음→" 버튼으로 수동 전환 방식 도입 (단, 원래 설계는 자동 전환이었음)

### 버그 B: Phase 2 터치 작동 여부
- **증상**: Phase 2에서 특수타일 스왑이 안 됨
- **원인**: 과거 `_pauseGame()`에서 `window.animating = true` 했었음 → 현재 코드에서는 제거됨
- **확인 필요**: 현재 코드로 Phase 2 터치가 실제로 작동하는지 검증 필요
